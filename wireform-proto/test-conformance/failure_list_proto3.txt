# Expected-failure list for the protobuf-conformance suite.
#
# Each line is either a comment (#) or a glob pattern matching
# the conformance_test_runner test names. Tests that match a
# pattern here are expected to fail; the runner's exit code
# reflects only unexpected failures (regressions) and unexpected
# successes (tests we should remove from this list).
#
# Pattern syntax is the runner's own; see
# https://github.com/protocolbuffers/protobuf/blob/main/conformance/conformance_test_runner.cc
# for the parser.
#
# Categories of expected failure:
#
# 1. JSON conformance for messages whose schema we don't include
#    (Well-Known Types). The handler returns Skipped for those.
# 2. JSPB tests — the handler always skips JSPB.
# 3. TEXT_FORMAT tests — not implemented yet.
# 4. Tests whose Required.* category we haven't yet covered.
#
# Update this file when the suite is run for the first time:
#
#     cabal test wireform-proto:protobuf-conformance-test 2>&1 | tee out.log
#
# Then add any newly-failing tests below, with a one-line comment
# explaining why we expect the failure.
