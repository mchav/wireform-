# Expected-failure list for the protobuf-conformance suite.
#
# Each line is either a comment (#) or a glob pattern matching a
# test name. Tests matching a pattern are expected to fail; the
# runner's exit code reflects only unexpected failures (regressions)
# and unexpected successes (tests we should remove from this list).
#
# Today's residue (203 failures from 1186 successes / 1286 skipped):
#
#   1. JSON for messages with Well-Known Types (Any, Wrapper,
#      Timestamp, Duration, FieldMask, Struct, Value, ListValue,
#      Empty). The handler skips these because the spliced
#      TestAllTypesProto3 omits WKT arms (loadProto doesn't yet
#      follow proto imports).
#
#   2. JSPB tests - the handler always skips JSPB.
#
#   3. TEXT_FORMAT tests - not implemented yet.
#
#   4. JSON-input parser edge cases we don't yet match:
#      - EnumFieldWithAlias / EnumFieldWithAlias{Different,Lower}Case
#      - FieldName{Escaped, InSnakeCase, WithDoubleUnderscores,
#        WithMixedCases, WithNumbers}: proto3 spec accepts both
#        camelCase and snake_case JSON keys; we only accept the
#        configured camelCase form.
#      - OneofZero{X}: JSON-input handling of oneof variants set
#        to their default value.
#      - {Float,Double,Int64}Field{TooLarge,TooSmall}: range
#        validation we don't perform.
#      - DoubleFieldTooSmall: same.
#      - MessageMapField: parsing maps whose values are
#        submessages.
#      - OneofFieldDuplicate / OneofFieldNullFirst / Second: tricky
#        oneof JSON merge semantics.
#      - BytesFieldBase64Url: base64url variant decoding.
#      - JSON Validator-class assertions on canonical output
#        details we don't yet emit (FieldNameInLowerCamelCase, etc.).
#
#   5. Submessage merge on repeated wire occurrence
#      (RepeatedScalarMessageMerge, ValidDataOneof.MESSAGE.Merge,
#      ValidDataOneofBinary.MESSAGE.Merge): proto3 spec says when a
#      singular submessage field appears multiple times on the
#      wire, the parser must merge them rather than overwrite. Our
#      decoder currently overwrites.
#
# Update this file when the suite is re-run on a new VM:
#   cabal test wireform-proto:protobuf-conformance-test \
#     2>&1 | tee /tmp/conformance.log
# then mine /tmp/conformance.log for new failure names.
