# Expected-failure list for the protobuf-conformance suite.
#
# Each line is either a comment (#) or a glob pattern matching a
# test name. Tests matching a pattern are expected to fail; the
# runner's exit code reflects only unexpected failures (regressions)
# and unexpected successes (tests we should remove from this list).
#
# Today's residue (119 failures from 1279 successes / 1277 skipped):
#
#   1. JSON-input parser edge cases:
#      - EnumFieldWithAlias / EnumFieldWithAlias{Different,Lower}Case
#      - FieldName{Escaped, InSnakeCase, WithDoubleUnderscores,
#        WithMixedCases, WithNumbers, OriginalProtoFieldName}: proto3 spec
#        accepts both camelCase and snake_case JSON keys; we only accept
#        the configured camelCase form.
#      - OneofZero{X}: JSON-input handling of oneof variants set to
#        their default value (parser doesn't yet route oneof variants
#        on input — the JSON has the variant key, but our schema
#        descriptor expects the carrier name).
#      - {Float,Double,Int64}Field{TooLarge,TooSmall}: range
#        validation we don't perform.
#      - DoubleFieldTooSmall: same.
#      - MessageMapField: parsing maps whose values are
#        submessages (only a few specific shapes work).
#      - OneofFieldDuplicate / OneofFieldNullFirst / Second: tricky
#        oneof JSON merge semantics.
#      - BytesFieldBase64Url: base64url variant decoding.
#      - JSON Validator-class assertions on canonical output details
#        we don't yet emit.
#
#   2. Two oneof-submessage-merge cases (the inner submessage carries
#      wire-type-mismatched fields; spec wants tolerant skip, our
#      decoder fails-strict in the merge path).
#
#   3. WKT JSON edge cases:
#      - FieldMaskNumbersDontRoundTrip / PathsDontRoundTrip /
#        TooManyUnderscore: the FieldMask path-encoding spec has
#        snake_case <-> camelCase rules we don't fully implement.
#      - DurationProtoInputTooLarge / TooSmall, TimestampProto
#        InputTooLarge / TooSmall: range validation on Duration /
#        Timestamp serialised values.
#      - Recommended.Proto3.Value{RejectInf,RejectNan}NumberValue:
#        Value-type rejecting NaN / Infinity number values.
#      - DurationHas[N]FractionalDigits.Validator,
#        TimestampHas[N]FractionalDigits.Validator: validator
#        assertions on canonical fractional-digit counts.
#      - Various Any.{X} tests: Any encoding requires a runtime
#        type registry we haven't wired in.
#
# Update this file when the suite is re-run on a new VM:
#   cabal test wireform-proto:protobuf-conformance-test \
#     2>&1 | tee /tmp/conformance.log
# then mine /tmp/conformance.log for new failure names.
